# انگور — پروژه کامل اندروید

این مخزن نسخه اندروید برنامه مدیریت مالی **Angoor / انگور** است. رابط برنامه از فایل HTML داخلی اجرا می‌شود و قابلیت‌های حساس موبایل با کد بومی اندروید پیاده‌سازی شده‌اند.

## گرفتن APK از GitHub

> **نکته آپلود در GitHub:** پوشه `.github` در ویندوز ممکن است مخفی باشد. قبل از کشیدن فایل‌ها داخل GitHub، گزینه **View > Show > Hidden items** را فعال کنید تا مسیر `.github/workflows/build-apk.yml` نیز آپلود شود.

1. تمام محتوای این پوشه را در ریشه یک مخزن GitHub قرار دهید.
2. وارد زبانه **Actions** شوید.
3. گردش‌کار **ساخت APK انگور** را باز کنید.
4. روی **Run workflow** بزنید؛ با هر Push روی شاخه `main` یا `master` نیز ساخت خودکار انجام می‌شود.
5. پس از سبزشدن Build، از بخش **Artifacts** فایل `Angoor-APK` را دانلود کنید.
6. داخل Artifact فایل `Angoor.apk` قرار دارد.

## ساخت در Android Studio

پروژه را در Android Studio باز کنید و پس از Gradle Sync از مسیر **Build > Build APK(s)** استفاده کنید، یا در خط فرمان اجرا کنید:

```bash
./gradlew lintDebug assembleDebug
```

خروجی دیباگ:

```text
app/build/outputs/apk/debug/app-debug.apk
```

## مشخصات پروژه

- شناسه برنامه: `ir.angoor.app`
- نام برنامه در لانچر گوشی: `Angoor`
- نسخه: `1.8.0` — کد نسخه `18`
- حداقل اندروید: Android 7.0 / API 24
- نسخه هدف و کامپایل: Android API 36
- Java: نسخه 17
- Android Gradle Plugin: نسخه 9.2.1
- Gradle Wrapper: نسخه 9.4.1
- رابط: WebView محلی، بدون دریافت فایل رابط از اینترنت

## قابلیت‌های بومی اندروید

- آلارم و اعلان سیستم با `AlarmManager` حتی هنگام بسته‌بودن برنامه و خاموش‌بودن صفحه
- نمایش تمام‌صفحه آلارم روی صفحه قفل، صدا، لرزش و دکمه تأیید
- بازیابی یادآورها بعد از روشن‌شدن گوشی، تغییر ساعت و به‌روزرسانی برنامه
- درخواست دسترسی آلارم دقیق در Android 12 و بالاتر و حالت جایگزین در صورت رد دسترسی
- بررسی و درخواست مجوز نمایش تمام‌صفحه آلارم در Android 14 و بالاتر
- رعایت حاشیه امن نوار وضعیت، بریدگی نمایشگر و نوار پیمایش با کوچک‌کردن واقعی محدوده WebView
- ورود با اثر انگشت از طریق AndroidX Biometric
- تشخیص گفتار فارسی از طریق سرویس بومی Android Speech Recognizer با مهلت ۱۰ ثانیه برای فرمان و ۲۰ ثانیه برای تأیید
- ذخیره PDF و فایل پشتیبان در پوشه اصلی `Downloads`؛ PDF پس از ذخیره اعلان می‌دهد و خودکار باز می‌شود
- اشتراک‌گذاری فایل پشتیبان با Share Sheet اندروید
- انتخاب فایل JSON برای بازیابی نسخه پشتیبان
- آیکون بنفش اصلی انگور با حاشیه امن و نسخه Adaptive بدون برش برای لانچرهای جدید

## مجوزهای مورد استفاده

- اینترنت برای سرویس تبدیل گفتار به متن
- میکروفن برای ثبت صوتی
- اعلان برای نمایش یادآورها
- آلارم دقیق برای اجرای یادآور در زمان ثبت‌شده
- اجرا پس از روشن‌شدن گوشی برای بازگردانی یادآورها
- لرزش و Wake Lock برای آلارم

کاربر هنگام ثبت اولین یادآور، مجوز «اعلان»، «آلارم‌ها و یادآورها» و در نسخه‌های جدید اندروید «اعلان تمام‌صفحه» را در خود سیستم تأیید می‌کند. اگر مجوز آلارم دقیق داده نشود، برنامه از زمان‌بندی تقریبی اندروید استفاده می‌کند.

## بررسی خودکار قبل از Build

این دستور ساختار پروژه، XMLها، JavaScript، آیکون‌ها و اتصال HTML به کد بومی را بررسی می‌کند:

```bash
python3 scripts/validate_project.py
```

GitHub Actions این بررسی را پیش از ساخت APK اجرا می‌کند. Lint پس از ساخت و به‌صورت غیرمسدودکننده اجرا می‌شود تا خطای Lint مانع دریافت APK نشود.

## امضای ثابت نسخه Release

بدون تنظیم Secret، GitHub یک APK دیباگ قابل نصب می‌سازد. برای انتشار نسخه Release و نصب به‌روزرسانی بدون حذف اطلاعات، یک keystore ثابت ایجاد کنید و این Secretها را در مسیر **Settings > Secrets and variables > Actions** قرار دهید:

- `ANGOOR_KEYSTORE_BASE64`
- `ANGOOR_KEYSTORE_PASSWORD`
- `ANGOOR_KEY_ALIAS`
- `ANGOOR_KEY_PASSWORD`

نمونه ساخت کلید:

```bash
keytool -genkeypair -v -keystore angoor-release.jks -alias angoor -keyalg RSA -keysize 2048 -validity 10000
```

تبدیل به Base64 در لینوکس یا macOS:

```bash
base64 < angoor-release.jks | tr -d '\n'
```

فایل keystore و رمزهای آن را داخل مخزن عمومی قرار ندهید.
